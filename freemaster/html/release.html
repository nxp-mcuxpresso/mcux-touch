NXP Touch v2.12

Overview

The release notes in this document are for the last NXP Touch software (NT) version. This is a full-function library to handle touch events measured by the NXP TSI, CS module, or GPIO methods. It also represents the measured results for various controls, such as a keypad, rotary, slider, analog slider, analog rotary, proxi, or matrix.

What is new

This release is the first public release, and it has the following features:

  • Supported devices
    • MCXN947 (TSIv6)
    • MKE17Z512 (2xTSIv5)
    • MKE17Z (2xTSIv5)
    • MKE16Z (TSIv5)
    • MKE15Z (TSIv5)
    • FRDM-MK32L2A4S (TSIv4)
    • QN9080 (CS)
  • Supported modules (peripherals/methods)
    • TSI v6
    • TSI v5
    • TSI v4
    • CS
  • Supported controls
    • Slider
    • Rotary
    • Analog Slider
    • Analog Rotary
    • Proxi
    • Matrix

NXP Touch release overview

NXP Touch (NT) is intended for use with the Kinetis MCU product family based on the Arm Cortex-M series architecture. The whole library is designed to work with the MCUXpresso SDK. The release consists of the following:

  • Documentation - Reference manual
  • Example application for the supported board
  • FreeMASTER application for NT with Touch GUI tool
  • NT library source code

Documentation

This release contains the generated documentation that describes the whole library with block schemes and the whole API is described either as public or all-private internal API. The documentation is provided in the html format.

Example application for supported boards

There are a couple of example applications that run on standard NXP development boards and show all aspects of NT capabilities. All applications are suitable for all supported development tools.

NXP Touch GUI Tool in FreeMASTER

The NXP Touch GUI Tool (Touch GUI Tool) and the FreeMASTER application are provided for better understanding and tuning of the final configuration values. The FreeMASTER automatically generates the HTML pages for visualization touch quantities in real time according to the current configuration of the embedded application using TSA tables.

NXP Touch library software

NT is distributed as a source code to allow you to check the code and extend it using your own modules/key detectors/controls to meet the specific needs. The code is divided into two main folders as follows:

  • Includes - All public include header files; this is the only folder that must be added to the preprocessor path for the NT library
  • Source - The source code and private header files for the NT library

Known issues

Project paths

All the project paths are set up during the installation as an absolute path, relative to the MCUXpresso SDK installation. This enables you to have the project installed on the PC in the folder where the NT is installed, but it can cause issues on any other PCs.
