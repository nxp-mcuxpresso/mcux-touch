CS Drivers
NXP Touch User API

Overview

Drivers represent the lowest level of abstraction in Capacitive sense (CS) peripheral control and contain common and CS version specific files. There are functions like NT_CS_DRV_Init, NT_CS_DRV_EnableElectrode, NT_CS_DRV_Measure, etc.

struct nt_cs_operation_mode_t

Driver operation mode data hold structure.

This is the operation mode data hold structure. The structure is keep all needed data to be driver able to switch the operation modes and properly set up HW peripheral.

Typedef Documentation

typedef void(* cs_callback_t)(uint32_t instance, void *usrData)

The function is called on end of the measure of the CS driver. The function can be called from interrupt, so the code inside the callback should be short and fast.

Parameters
instance- instance of the CS peripheral
usrData- user data (type is void*), the user data are specified by function NT_CS_DRV_SetCallBackFunc
Returns
- none
