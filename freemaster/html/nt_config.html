Configuring the Library

This section describes the library structures that must be initialized to use the NXP Touch library in an application. Almost all library-configuration parameters are passed to the library API in one of the System, Modules, keydetectors, Electrodes, or Controls structures. The same structures that are used to keep the configuration data for the library initialization are then used in the application code to access their run-time properties.

For example: the user initializes the nt_aslider_control structure to specify the set of electrodes making up the slider layout, the maximum possible resolution of the slider, and other options. In the application run-time, the code uses a pointer to the same structure to access the calculated values (such as the finger position).

Modules

This object depends on the type of a module. This structure binds together electrodes assigned to module and other parameter (watermarks, autocalibration, safety, instance, ...) depending on module. The module setup is displayed below (including the hardware configuration). Firstly all module electrode must be assigned.

/* Modules */
const struct nt_electrode * const module_electrodes[] = { \
&El1, &El2, /* FRDM-KE15Z electrodes */ \
&El3, &El4, &El5, &El6, /* FRDM-TOUCH electrodes */ \
&El7, &El8, /* analog slider electrodes */ \
&El9, &El10, &El11, &El12, /* analog rotary electrodes */
NULL};

CS module

Hardware configuration of CS module is shown below.

/* Self-cap config */
const cs_config_t tsi_hw_config =
{
/* For acitve mode */
.activeClockDivider = 99U, /* Active mode clock divider, occupy 9 bits, generates CLK_CS clock from CLK_APB clock using F(CS) = F(APB) / (clockDivider+1) */
.activeDetectPeriod = 350U, /* Detect period (how many CLK_CS clock cycles to scan a channel, occupy 16 bits */
.activeIdlePeriod = 5U, /* Idle period (how many clock cycles to wait between channel scans), occupy 16 bits */
.activeOscFreq = 63U, /* Relaxation oscillator frequency configuration. The value will adjust the current being driven to input channels by equation: 0.4 + 0.2(OSC_FREQ+1) [uA], occupy 6 bits */
.activeCounterGain = 25U, /* Can increase counter value by multiplication with this gain, cccupy 8 bits */
/* For low power mode */
.lowPowerChannelNum = 4U, /* Indicates how many consecutive CLK_32K cycles need to meet the threshold value before the wake-up interrupt is triggered. The actual interpretation by the hardware of this value is such that one more than the value programmed here is used, occupy 4 bits */
.lowPowerThreshold = 3150U,/* The value represents the upper bound of the counter output in terms of what is considered a valid touch event, occupy 16 bits*/
.lowPowerDebonceNum = 50U, /* Selects which channel index to monitor while in LP mode, occupy 3 bits*/
.lowPowerIdlePeriod = 20U, /* Idle period between previous scanning end and next scanning start, occupy 16 bits */
.lowPowerOscFreq = 30U, /* OSC bias select, occupy 6 bits */
};

For operation with nt_module, see the code below.

const struct nt_module module =
{
.interface = &nt_module_cs_interface, /* Module interface. Can't be NULL. */
.electrodes = &module_electrodes[0], /* A pointer to the list of electrodes. Can't be NULL. */

.wtrmark_lo = 0, /* signal too low watermark */

.safety_interface = &nt_safety_interface, /* An instance module Safety params. NULL when safety not needed */
.safety_params.gpio = &my_safety_params, /* Module Safety interface. NULL when safety not needed */
.recalib_config = (void*)&recalib_config, /* Recalibration structure for autocalibration */

System

The System is represented by the nt_system structure. This structure binds together all the other objects, so it must be initialized with the following parameters:

  • list of controls
  • list of modules (which indirectly provides the list of all electrodes used)
  • time period and initialization time
  • safety configuration (multiplication of time period, HW CRC and interface
